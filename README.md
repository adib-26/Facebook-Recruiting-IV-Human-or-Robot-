# Facebook-Recruiting-IV-Human-or-Robot-

> **Kaggle Competition:** [Facebook Recruiting IV: Human or Robot?](https://www.kaggle.com/competitions/facebook-recruiting-iv-human-or-bot)  
> Predict whether an online auction bid was placed by a human or an automated bot.

---

##  Overview

Online auction platforms face a growing threat from automated bidding bots that crowd out human participants. This project tackles that problem using a **Hybrid Neural Network (HybridBotNet)** built in PyTorch, which fuses two complementary detection strategies:

- **Temporal Dynamics** — a 2-layer LSTM captures the rhythm and cadence of sequential bids over time.
- **Structural Fingerprints** — a deep feedforward network processes aggregated behavioural statistics like IP diversity, device switching, and bidding concurrency.

The two pathways are fused at the classification head to produce a final bot-probability score.

##  Dataset

| `bids.csv` | Bid event log — `bidder_id`, `auction`, `time`, `ip`, `device`, `url`, `country` |

> **Class Imbalance:** The dataset is heavily skewed toward human bidders. The model compensates with a weighted `BCEWithLogitsLoss` using an adjusted imbalance ratio (`human/bot × 1.5`).

##  Feature Engineering

Raw bid logs are transformed into two feature families fed to the two network branches.

### Sequential Features (LSTM input)
Each bidder's bid history is represented as a fixed-length sequence of `MAX_SEQ_LEN = 50` log-scaled inter-bid time intervals:

```
log_time_diff = log1p(time[i] - time[i-1])
```

Sequences are **right-padded with zeros** for bidders with fewer than 50 bids and **truncated to the last 50** for those with more.

### Static / Aggregated Features (FFN input)

| Feature Group | Features |
|---|---|
| **Time Statistics** | `mean_diff`, `std_diff`, `min_diff`, `median_diff`, `max_diff` of `log_time_diff` |
| **Concurrency** | `max_concurrent`, `mean_concurrent` (bids at the exact same microsecond) |
| **Entity Counts** | `num_bids`, `num_auctions`, `num_devices`, `num_ips`, `num_urls`, `num_countries` |
| **Behavioural Ratios** | `bids_per_auct`, `bids_per_device`, `bids_per_url`, `ip_per_ctry` |

All static features are normalised with `StandardScaler` fitted on the training set.

##  Model Architecture — HybridBotNet

```
Input: Sequential bid history (batch, 50, 1)      Input: Static features (batch, N)
            │                                                    │
    ┌───────▼────────┐                              ┌────────────▼───────────┐
    │  LSTM (×2)     │                              │  Linear → ReLU → Drop  │
    │  hidden=128    │                              │  Linear → ReLU         │
    │  dropout=0.4   │                              └────────────┬───────────┘
    └───────┬────────┘                                           │
            │  last hidden state                                 │
            └──────────────────────┬────────────────────────────┘
                                   │  Concatenate (128 + 64 = 192)
                              ┌────▼────┐
                              │ Linear  │
                              │  ReLU   │
                              │  Drop   │
                              │ Linear  │
                              └────┬────┘
                                   │  Logit → Sigmoid → Bot Probability
```

##  Results

### Training Performance
| Metric | Score |
|---|---|
| Final Training AUC | ~94% |
| 5-Fold Cross-Validation AUC | ~86.5% ± σ |

### Cross-Validation (5-Fold Stratified)
Cross-validation confirms the model generalises beyond the training set. Per-fold AUC scores are visualised via a bar chart with the mean marked as a reference line.

### Operational Threshold
A threshold of **0.70** was selected after evaluating the precision/recall tradeoff across a range of thresholds (0.10 → 0.95, step 0.05). At this threshold:
- Bot detection recall is prioritised over false-alarm minimisation.
- The confusion matrix is visualised in the notebook.

##  Key Design Decisions

- **Log-scaling time intervals** prevents LSTM gate saturation caused by the wide dynamic range of raw microsecond timestamps.
- **Class-weight boosting** (`×1.5` beyond the raw imbalance ratio) forces the model to be more aggressive in flagging bot behaviour.
- **Outlier removal** (5 flagged indices) improves gradient stability during early training.
- **K-Fold cross-validation** guards against overfitting and provides a statistically reliable AUC estimate.
